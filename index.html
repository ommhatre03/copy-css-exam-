Question 2

Practical Question: Create JSON Object Person with name, age and city with default values. Display them.

Theory Question: How do you convert JSON object into javascript object?

Answer

Working Code:

<!DOCTYPE html>
<html>
<head>
    <title>JSON to JavaScript Object</title>
</head>
<body>
    <h3>Person Details:</h3>
    <p>Name: <span id="name"></span></p>
    <p>Age: <span id="age"></span></p>
    <p>City: <span id="city"></span></p>

    <script>
        // 1. A JSON string representing a "Person" object
        const personJSON = `{
            "name": "John Doe",
            "age": 30,
            "city": "New York"
        }`;

        // 2. Convert the JSON string into a JavaScript object
        const personObject = JSON.parse(personJSON);

        // 3. Display the values in the HTML
        document.getElementById('name').textContent = personObject.name;
        document.getElementById('age').textContent = personObject.age;
        document.getElementById('city').textContent = personObject.city;
    </script>
</body>
</html>

Live Demo:

Theory Answer:

You convert a JSON string into a JavaScript object using the built-in JavaScript function JSON.parse().

Syntax: JSON.parse(jsonString)

  • jsonString: A string that contains valid JSON-formatted data.
  • Return Value: It returns a new JavaScript object that corresponds to the data in the JSON string.

It's important to note that what is often colloquially called a "JSON object" is technically a "JSON string representing an object". The term "JSON object" is just shorthand. The conversion process takes this string and materializes it into a real, usable JavaScript object in memory.

The reverse operation, converting a JavaScript object into a JSON string, is done using JSON.stringify().

Question 3

Practical Question: Set the first name and last name in separate property and display full name.

Theory Question: What is interpolation in data binding?

Answer

Working Code:

<!DOCTYPE html>
<html ng-app="myApp">
<head>
    <title>AngularJS Interpolation</title>
    <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.2/angular.min.js"></script>
</head>
<body ng-controller="myCtrl">
    <h3>Full Name Display</h3>
    <p>Full Name: <strong>{{ firstName + ' ' + lastName }}</strong></p>

    <script>
        var app = angular.module('myApp', []);
        app.controller('myCtrl', function($scope) {
            $scope.firstName = 'Jane';
            $scope.lastName = 'Doe';
        });
    </script>
</body>
</html>

Live Demo:

Theory Answer:

Interpolation is a form of one-way data binding that embeds expressions into the HTML template. In AngularJS, it is denoted by double curly braces: {{ expression }}.

  • Function: It binds an expression from the controller's $scope to the view (the HTML). When the value on the $scope changes, AngularJS automatically updates the text in the HTML.
  • One-Way Binding: It's called one-way binding because data flows in one direction only: from the model ($scope) to the view. Changes in the view do not affect the model.
  • Usage: It's primarily used to display data, such as text, numbers, or the result of simple expressions, directly in the HTML content.

Theory Answer:

The key difference between one-way and two-way data binding lies in the direction of data flow between the model (data and logic) and the view (UI).

Feature One-Way Binding Two-Way Binding
Data Flow Unidirectional: Model ➔ View Bidirectional: Model ⇄ View
Description Changes in the model automatically update the view. However, changes in the view (e.g., user input) do not affect the model. Changes in the model update the view, AND changes in the view (e.g., user input in a form field) automatically update the model.
AngularJS Example {{ expression }} (Interpolation) or ng-bind ng-model (used on form elements like input, select, textarea)
Use Case Displaying data that the user doesn't directly edit (e.g., titles, labels, read-only information). Forms and user input fields where the application needs to react instantly to user changes.
